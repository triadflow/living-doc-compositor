Stage Skill Matrix

A living doc stage is the current condition of the document as a working system. The stage is inferred from objective clarity, convergence-type structure, proof state, source-system evidence, and tension between those parts.

Core reading

  • Stages are not decorative status labels.
  • Each stage has a different valid skill response.
  • Chemistry skills mainly govern unstable transformations.
  • Early shaping skills make later execution possible.

Chemistry Alignment

The chemistry concept is strongest when a living doc is changing state. These laws describe what a skill must protect during the transformation.

Conservation Objective mass cannot disappear.

Every accountable objective term must reappear as complete, blocked, deferred, pivoted, out of scope, or unresolved.

Activation energy Closure needs a real threshold.

Passing tests, rendered pages, or closed tickets are not enough unless they prove the objective.

Equilibrium State surfaces must agree.

The document, source system, issues, proof, and user feedback cannot point to different truths.

Catalysis A skill lowers repair cost.

The skill accelerates the right transformation without changing what must be conserved.

Reaction path Not every stage jump is valid.

A useful slice cannot jump straight to final closure while objective-owned work remains unaccounted.

How To Use The Matrix

Use the matrix to choose the next skill from the living doc's current condition, not from the artifact you hope it has become.

If the objective is soft

Run objective-acceptance-shaping. Do not start by checking readiness when the objective itself still cannot close.

If the doc looks ready

Run objective-execution-readiness. The question is whether a later agent can tell what remains false.

If closure feels slippery

Run the chemistry skills. First classify the imbalance, then conserve objective terms before accepting closure.
