What lets Codex move independently.

Independence does not come from asking the model to take more initiative. It comes from a living doc that turns the objective into source-verifiable acceptance criteria, so the next correct move is visible.

Rene Luijk Triadflow
DraftedMay 7, 2026 Read4 min StatusDraft

Codex moves independently when the next valid action is visible in the work surface. The model does not need more freedom. It needs fewer hidden interpretations.

That happens when the objective is literal, the acceptance criteria can fail, and each open term still points to a source-system proof.

The convergence-type refactor made this visible. Once the objective was fixed verbatim, the remaining work could be sliced by accountable terms. The living doc made the gates visible: code-defined convergence types, deterministic registry generation, starter-template composition, rendered-doc preservation, MCP contract access, and drift-prevention tests.

Each gate could point at a concrete proof: a file path, a generator command, a rendered artifact, an MCP response, a contract test, or an issue that still needed to remain open.

That is why the work could continue without asking for a new decision at every step. The next action was not chosen from taste. It was chosen from what was still false against the objective.

literal objective
+ acceptance criteria that can fail

The objective has to be literal

A useful objective is not a theme. It is a contract. It names the system change, the source surfaces involved, and the condition under which the work can close. If the wording can be reinterpreted after a partial implementation, it is not strong enough yet.

Weak objective

Improve the living-doc semantic source so tools can reason better.

Executable objective

Refactor the system so convergence types are the executable canonical source, the registry is generated from them, starter templates compose them, MCP can query their contracts, and tests prevent drift across generated and runtime surfaces.

The second version gives Codex room to act because it also gives Codex less room to pretend. Each phrase becomes a gate.

Acceptance criteria are the rails

Acceptance criteria should not be a polite checklist. They should be closure gates that can fail. A good criterion points to a source-system proof: a generator check, a contract test, a rendered artifact, an MCP response, an issue state, or a living-doc section that records the remaining truth.

  1. Name the accountable terms.Break the objective into the words that cannot disappear.
  2. Attach proof to each term.Define what command, artifact, issue, or rendered state proves that term.
  3. Keep partial slices partial.A ticket can close while the objective remains open.
  4. Make drift detectable.Ask what should fail if the implementation and the objective diverge.
  5. Update the living doc as work changes.The doc must tell the next session what remains false.

The agent becomes less dependent

This is the practical effect: Codex can continue because the living doc gives it an operating surface. It can inspect the current objective, see which acceptance criteria remain unsatisfied, map work to issues, run the proof commands, update the doc, and know when the closure claim is still premature.

That is not autonomy as personality. It is autonomy as constrained execution. The living doc carries intent, proof, state, and closure gates in a form the agent can keep using.

What independence does not mean

Independent execution is not permission to choose a new objective, skip source evidence, or close the work because the local implementation feels coherent. It means Codex can keep moving inside a boundary that the living doc already made explicit.

The short rule

When you want Codex to move more independently, do not start by giving it a bigger instruction. Start by making the objective less negotiable and the acceptance criteria more testable.

Then Codex does not have to guess what initiative means. It can inspect what remains false, make one of those things true, and leave the living doc clearer for the next run.
