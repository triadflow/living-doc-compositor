Living Doc Harness Inference Chain

The lifecycle controller does not reason for the system. It starts separate inference actors, passes file paths, records artifacts, and enforces whether the next step may run.

The core idea is simple: every meaningful judgment is its own inference call. Worker inference does work. Reviewer inference reads the worker JSONL logs. Balance scan inference orders repair skills. Each repair skill runs as another inference call.

Worker Inference

Moves the objective forward and leaves raw JSONL logs.

Reviewer Inference

Opens raw worker JSONL paths and emits the stop verdict.

Repair Inference

Runs balance scan and ordered repair skills as independent actors.

Lifecycle Control

Chains calls, validates artifacts, renders docs, and stops on blockers.

  E -->|repairable| I["Repair phase"]
  I --> J["Balance scan inference"]
  J --> K["Ordered repair skill list"]
  K --> L["Repair skill chain"]
  L --> M["Patched + rendered living doc"]
  M --> N["Reviewer / readiness check"]
  N --> G

The controller is only allowed to continue after the required inference actor has left durable evidence.

Repairable Verdict Path

sequenceDiagram
  participant L as Lifecycle
  participant R as Reviewer Inference
  participant B as Balance Scan Inference
  participant S as Repair Skill Inference
  participant D as Living Doc

  R-->>L: verdict = repairable
  L->>B: doc path + verdict path + handover path + raw worker JSONL paths
  B->>B: classify imbalance
  B-->>L: ordered repair skill list
  loop for each ordered skill
    L->>S: skill name + updated doc + prior results + raw worker JSONL paths
    S->>D: inspect and patch if needed
    S->>D: render if changed
    S-->>L: result JSON + skill JSONL log
    L->>L: validate result before next skill
  end
  L-->>R: repaired doc evidence for next review

Each Skill Result Must Say

  • status: repaired, no-op, blocked, failed
  • changedFiles: concrete paths
  • renderedHtmlPath: required after doc changes
  • priorRepairResultPaths: what it used
  • rawJsonlLogPath: its own inference log
  • nextRecommendedAction: continue, block, review

Stop Rules

  A["Skill result"] --> B{"status"}
  B -->|repaired| C["render if changed
continue chain"]
  B -->|no-op| D["record no-op
continue chain"]
  B -->|blocked| E["write blocker
stop chain"]
  B -->|failed| F["write process defect
stop chain"]
  C --> G["next ordered skill or review"]
  D --> G

A recommended skill list is not proof. Only completed skill-run artifacts can move the lifecycle forward.
