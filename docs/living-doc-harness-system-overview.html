What The System Does

The harness turns a living doc into the control surface for a headless Codex run. The worker can edit source files and the living doc, but closure is decided outside the worker from native trace refs, objective state, proof gates, terminal state, and sanitized evidence artifacts.

The Two Loops

Worker Loop

Codex runs headless against one living doc objective. It performs the implementation work, updates files, may update the living doc, and emits wrapper events plus native trace files. This loop is allowed to make progress, but it is not allowed to certify its own completion.

Governance Loop

The harness inspects what happened after the worker stops. It reads trace summaries, compares proof against objective terms and acceptance criteria, classifies the stop, routes repair skills when needed, and only then writes terminal state.

Runtime Loop

1. Living doc
Objective, success condition, acceptance criteria, invariants, issue spine, and proof ladder.
2. Standalone runner
Creates a run directory and launches codex exec --json outside the active user session.
3. Native trace
Discovers Codex JSONL traces, writes sanitized summaries, and attaches trace refs to the run.
4. Stop diagnosis
Infers whether the stop is repairable, resumable, closed, blocked, pivoted, deferred, or budget-exhausted.
5. Skill routing
Maps verdicts to living-doc skills and records repair handovers or closure audits.
6. Finalizer
Writes evidence, verdict, terminal state, proof validation, bundle, and dashboard state.
7. Dashboard
Shows run state, gates, blockers, trace summaries, evidence artifacts, and recommendation.

Component Ownership

docs/living-doc-agentic-harness.json

scripts/living-doc-harness-runner.mjs
Creates standalone run directories and launches the worker through codex exec --json. In execute mode it also discovers native trace files and attaches sanitized summaries.
scripts/living-doc-harness-trace-reader.mjs
Finds Codex JSONL rollout traces, creates privacy-safe summaries, hashes the trace evidence, and records trace references without committing raw content.
scripts/living-doc-harness-stop-negotiation.mjs
Classifies why the run stopped by comparing wrapper state, native trace refs, objective terms, acceptance criteria proof, blockers, and remaining work.
scripts/living-doc-harness-skill-router.mjs
Turns the stop verdict into concrete next actions: repair skills, final audits, resume instructions, blocker records, or closure recommendation.
scripts/living-doc-harness-iteration.mjs
Finalizes one iteration by writing evidence, stop verdict, skill handover, terminal state, proof validation, evidence bundle, and dashboard data.
scripts/living-doc-harness-evidence-dashboard.mjs
Builds the committed sanitized proof surfaces: evidence bundle, evidence summary, and static dashboard.

Control Layer

  • Objective and success condition stay in the living doc.
  • Governance fingerprint guards section/card structure.
  • Acceptance criteria carry proof requirements and current proof.
  • Issue spine keeps GitHub state tied to objective state.

Inference Layer

  • Worker runs headless through codex exec --json.
  • Worker output is evidence, not authority.
  • Native traces are summarized, hashed, and kept private.
  • Stop reason is inferred by the harness after the run.

Proof Layer

  • Iteration proof must validate before closure.
  • Evidence bundles omit raw prompts, logs, traces, and message content.
  • Dashboard provides the operator inspection surface.
  • Terminal states prevent silent continuation after true blocks.

Privacy Boundary

The committed proof surfaces are sanitized. Raw prompts, wrapper logs, native trace payloads, and message content stay out of the evidence bundle and dashboard. Raw local run logs remain in .living-doc-runs/ and are intentionally not committed.

rawPromptIncluded=falserawWrapperLogIncluded=falserawNativeTraceIncluded=falserawMessageContentIncluded=false

Operator Commands

The runner and finalizer are deliberately separate so the worker can run headless and the governance loop can inspect the evidence after it stops.

node scripts/living-doc-harness-runner.mjs start docs/living-doc-agentic-harness.json \
  --runs-dir .living-doc-runs \
  --execute \
  --now 2026-05-07T11:35:00.000Z

node scripts/living-doc-harness-iteration.mjs evidence-template \
  .living-doc-runs/<run-id>

node scripts/living-doc-harness-iteration.mjs finalize \
  .living-doc-runs/<run-id> \
  --iteration 1 \
  --evidence .living-doc-runs/<run-id>/artifacts/iteration-1-evidence.json

How Future Runs Use It

For a new living doc, the same harness starts from that document's objective, runs Codex headless, attaches native trace summaries, scaffolds iteration evidence, finalizes the stop verdict, and renders the dashboard. Closure is never inherited from this run; each living doc instance must satisfy its own proof gates.
