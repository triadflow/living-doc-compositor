Living Doc Harness Control Loop

A standalone internal visual for the lifecycle harness as a controller-owned loop over a living doc. The diagram separates runtime control from editorial explanation and grounds the system in code, contracts, tests, dashboard behavior, and run artifacts.

Created 2026-05-12 Standalone HTML No external dependencies Mermaid source included below

Primary Visual

The visual now has one main path: controller, worker, evidence, reviewer, router. The router has three outcomes: loop back through continue/repair, pass through closure review to closed, or exit by explicit user stop.

What The Visual Must Teach

  • The harness is a loop, not a linear pipeline.
  • The living doc objective is the operated state.
  • Worker output is evidence, not closure authority.
  • Only objective proof or explicit user stop exits the loop.

Source Grounding

  • living-doc-harness-lifecycle.mjs owns loop orchestration.
  • living-doc-harness-iteration.mjs finalizes evidence and selection.
  • living-doc-harness-terminal-state.mjs defines terminal semantics.
  • living-doc-harness-dashboard-server.mjs exposes graph and inspector surfaces.

Visual Rules

  • Blue arrows are normal controller flow.
  • Amber/red arrows are non-closure continuation paths.
  • Green exit is objective-proven closure.
  • Gray exit is explicit user stop.

Mermaid Source

This is the maintainable model behind the SVG. It is included as source text rather than rendered by an external Mermaid runtime.

flowchart LR
  Doc["Living Doc Objective\nobjective + state + proof + rendered surface"]
  Controller["Controller\nselects next unit"]
  Worker["Worker\ndoes work"]
  Evidence["Evidence Snapshot\nfreezes facts"]
  Reviewer["Reviewer\nclassifies state"]
  Router["Router\nchooses next"]
  Continue["Continue / Repair\nnon-closure states"]
  ClosureReview["Closure Review\nchecks terminal closure"]
  Closed["Closed\nobjective proven"]
  Stopped["User Stopped\nexplicit stop"]

  Doc -. state read / changed .-> Controller
  Controller --> Worker
  Worker --> Evidence
  Evidence --> Reviewer
  Reviewer --> Router
  Router -->|repairable / true-block / missing proof| Continue
  Continue --> Controller
  Router -->|closure candidate| ClosureReview
  ClosureReview -->|approved| Closed
  ClosureReview -->|blocked| Continue
  Router -->|explicit user stop| Stopped
