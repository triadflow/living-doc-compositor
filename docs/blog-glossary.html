By concept

Recurring terms across the corpus, with the articles where each one shows up. A reading aid more than a strict dictionary — some concepts appear under different names in different pieces; this index collapses them.

act / surface
Two kinds of convergence types. Act cards record a thinking-action just performed; surface cards reflect state that exists somewhere else. The disambiguating test: if the section stopped being maintained, would the underlying state still change in the world? In: Act and surface · Selera, jadi permukaan / Taste, made into a surface
convergence type / registry
The vocabulary of section kinds — tiny-experiment, decision-record, taste-signature, capability-surface, and ~40 more. Each carries promptGuidance, structural contracts, status sets. The registry is the language; the current refactor makes those reusable contracts executable as code rather than only JSON vocabulary. In: Act and surface · The registry is a language · Convergence types as executable semantic source · Selera, jadi permukaan / Taste, made into a surface
executable semantic source
The source-of-truth split introduced by the convergence-type refactor: a living doc instance stays JSON because it is state; a convergence type becomes code because it is a reusable semantic contract. Generated registry artifacts, rendered context, MCP contract access, skill reasoning surfaces, and drift checks should all derive from that source. In: Convergence types as executable semantic source
inference-time harness
A control surface around inference. At the skill level, it holds the right abstraction while the model thinks. At the lifecycle level, it starts contract-bound inference units, captures their artifacts, and routes the next unit until the living-doc objective is proven or the user explicitly stops the run. In: Guidance is an inference-time harness · Physical AI and Living Docs · The A-to-Z run skill · What lets Codex move independently · A living doc is the substrate, not the report
living doc / portable artifact
A self-contained, navigable projection of entities and their relationships. Travels as one HTML file. Higher-order abstraction that compresses the search space for humans and LLMs. In the standalone harness, it also becomes the operated substrate: the shared objective, proof, stage, and acceptance surface each inference unit reads or changes. In: The artifact and the index · What a living doc reveals · Building a living doc from context · A living doc works better with a door · A living doc is the substrate, not the report
contract-bound inference unit
A bounded model run with a specific role: worker, reviewer, balance scan, repair skill, closure review, or post-flight summary. It receives a prompt plus an input contract with artifact paths, including the living doc and relevant inference logs, then emits artifacts and a verdict the controller can route. In: A living doc is the substrate, not the report
contract handoff
The arrow between inference units in the lifecycle dashboard. A handoff is not just a visual edge: it carries gate, effect, status, artifact paths, verdict, and, when relevant, commitIntent evidence. The controller uses the contract to route without becoming the semantic owner of the objective. In: A living doc is the substrate, not the report
knowledge graph / index
The other half of the substrate quadrant. Where the living doc is a portable artifact, the index is queryable structure across many docs. The two compose; they don't compete. In: The artifact and the index
skill
A small specialized agent invoked by name (/tiny-experiment, /design-system, /dossier). Thin discovery + dialog layer over the registry types it serves. In: Guidance is an inference-time harness · The A-to-Z run skill · Skills that rebalance a living doc · Selera, jadi permukaan
false closure / repair skill
False closure happens when an agent treats green tests, rendered pages, closed tickets, or plausible proof cards as permission to narrow the objective. Repair skills read objective, proof, source state, and issues together so invalid closure is blocked and the next valid operation stays visible. In: Skills that rebalance a living doc
objective-bound acceptance
The mechanism that lets Codex move without repeated interpretation. The objective is literal, acceptance criteria can fail, source-system proof remains attached, and the living doc says what remains false. Independence means constrained execution inside that boundary, not permission to choose a new objective. In: What lets Codex move independently · Convergence types as executable semantic source
MCP doorway
A local Model Context Protocol server that lets agents enter a living-doc surface at the right depth — overview, structure, diagnostics, valid operations — while the artifact remains portable. In: A living doc works better with a door · Selera, jadi permukaan (Figma MCP at inference time)
pointer, not copy
Discipline for entity cards. A primitive (token, component, motif) reference where it lives — Figma variable, tokens.json key, repo path — never the value. The card is a navigable reference; the truth stays in the canonical source. In: Selera, jadi permukaan / Taste, made into a surface
taste-signature / design system types
Four convergence types added in the design-system category: design-system-surface, taste-signature, brief-to-system-alignment, design-system-derivation. The first concrete application of the act/surface split to a new domain. In: Selera, jadi permukaan / Taste, made into a surface
archaeology tax
The cost paid every time someone re-reads an issue's full history to make sense of it. A living doc collects that cost into something a fresh session can load — the tax is paid once. In: Skipping the archaeology before real work starts
tracker / topic page
A single-page running picture of one topic, refreshed on a cadence (weekly for fast topics, half-yearly for slower ones). Reader downloads a working tracker and forks for their own market. In: Six AI labs, one page · One page per topic you follow
onboarding / preparation
Practical use cases: building a private living doc when joining a new org (onboarding), or when arriving at a serious conversation with a real model of the other side (preparation). In: The fastest onboarding doc · Come prepared to the table
representation first
The shared move underneath physical-AI tools and the inference-time living-doc harness. Both turn unbounded domains into governed, code-like substrates before automation. Representation is what makes automation safe. In: Physical AI and Living Docs · The registry is a language
agreement bottleneck
As AI capability rises, the constraint on complex work shifts. The hard part stops being execution and becomes agreeing on what to execute — a social problem, not a technical one. Living docs are an agreement substrate. In: Agreement is what's left
