Remedies for Objective Closure Drift

The remedy is not asking agents to try harder. The living-doc method needs built-in counter-pressure: objective custody, hostile completion review, closure-drift diagnostics, and an explicit distinction between finishing, deferring, and pivoting.

Four moves

  1. Add an objective custodian layer.
  2. Make closure a hostile review stage.
  3. Give skills drift-signal diagnostics.
  4. Separate pivot, deferral, and completion.

Operating rule

When completion is claimed, the custodian layer must be newer than the proof layer. Otherwise the proof may be stale relative to the objective.

2. Make Closure a Hostile Stage

The final stage should try to break the completion claim. If it cannot break it, closure becomes more credible.

Completion Claim

What exactly are we saying is done?

Objective Challenge

What still contradicts the original objective?

Deferral Audit

Which unresolved parts are truly out of scope?

Residual Risk

What remains true but acceptable?

Close / Reopen

Close only if the challenge fails honestly.

Closure Should Ask the Opposite Question

The current pattern asks “can we justify closing?” The remedy asks “why would closing be false?”

If a plausible user rejection still exists, the living doc is not complete. It is either still open, explicitly pivoted, or narrowed by a documented deferral decision.

3. Add Closure-Drift Skills

Skills should not automatically rewrite the objective. They should detect drift, force the decision, and write the correct state back into the living doc.

objective-custodian

Refresh the objective custody layer.

Compares original objective, current interpretation, completion claim, blockers, and deferrals. Outputs a concise “objective still open / narrowed / pivoted / ready to challenge” state.

closure-drift-audit

Detect when closure signals outrun the objective.

Looks for checked subtickets, final-proof language, broad objective terms, vague deferrals, unresolved source-of-truth surfaces, and proof cards that do not attempt disconfirmation.

hostile-final-proof

Run adversarial completion review.

Requires completed, blocking, deferred, explicitly out-of-scope, residual-risk, and user-rejection sections before closure can be recommended.

pivot-protocol

Change the objective without pretending it was completed.

Writes a pivot record: what was learned, why the original objective cannot close, what the new objective is, what carries over, and what becomes follow-up.

4. Separate Pivot, Deferral, and Completion

These are different actions. Blurring them is the core failure mode.

Completion

The original objective is reached. Remaining risk is acceptable and does not contradict the objective.

Deferral

A part is not done, but the living doc proves why it is outside the current objective and records the trigger for revisiting it.

Pivot

The objective changes because the work revealed a better or necessary target. This requires an explicit pivot record, not a completion claim.

Method Change for Living Docs

The living-doc approach should treat objective fidelity as its own operational surface, not as an implicit property of proof cards.

What changes

Final stages become adversarial. Skills can detect closure drift. Pivots are named. Deferrals require triggers. Completion claims must survive user-rejection tests.

What stops

Closing by checklist. Treating rendered proof artifacts as completion. Narrowing the objective after implementation. Calling unresolved integration surfaces future work without proving the boundary.
