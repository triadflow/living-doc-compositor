Onboarding — zero-config landing

On first open, the app detects role assignments from subscriptions.json, explains what packages the user inherits, and drops them straight into a pre-populated feed. No setup wizard.

Key behavior

  • Role (and any personal packages like stakeholder-gertjan) come from team-curated assignments — user does not pick them.
  • The description each package carries is surfaced here verbatim, so the user sees what they were auto-signed-up for.
  • One tap to enter the feed; refinement happens by using the app, not by configuring it first.

Feed — the attention surface

Every card is an architectural claim plus its evidence plus inline refinement. Cards carry a visual marker showing whether they came from an inherited role package, a user-added subscription, or a pinned/elevated choice. Regressions and blockers sort above progress; digests aggregate on a per-package window.

Card anatomy

  • Origin strip (left edge): grey = inherited from role, blue = user-added, amber = user-pinned.
  • Header: project · block-name · audience label · timestamp.
  • Transition: from-status → to-status with tokens matching the living doc palette.
  • Evidence (diff-backed, the grounding arbiter) and Intent (session memory, why).
  • Grounding warning surfaces asymmetry when session intent exceeds diff evidence.
  • Inline actions: See diff · Watch · Mute · Disagree. One tap each.

Block detail — claim & conversation

Tap any event to drill into the block. Current status with evidence; history trail of all transitions; watcher count (aggregate, no names); user's own event-type toggles; and the disagreement affordance that turns broadcast into dialogue.

Two states shown

  • Consensus: single accepted claim, user can watch specific event types.
  • Contested: a watcher disagreed; block marked everywhere with contest banner until the next grounded commit resolves it.

My map — self-audit & refinement signal

The visualization of the user's own subscription surface. Shows every block they currently receive, grouped by project, with an explicit distinction between inherited (from role), chosen (user-added), pinned, and muted. The app gently nudges toward refinement when a user is running entirely on defaults.

What this screen does

  • Answers "what am I actually watching these days?" at a glance.
  • Makes inherited-vs-chosen visually obvious — corrodes the silent-watcher problem.
  • Surfaces aggregate signals privately to the user (e.g., "you rarely open this block — unwatch?").
  • Bulk-edit entry point: tap a row to toggle, long-press for event-type config.
