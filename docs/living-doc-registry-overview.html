How to read this page

Each type is a formal reasoning contract, not a visual template.

The overview reads the current local living-doc catalog and shows real sections implementing each type. That makes it easier for humans and LLMs alike to see the implementation level instead of reasoning from the abstract alone.

1 Description

What kind of thing the type is. One sentence; unambiguous about the convergence it claims.

2 Structural contract

What a valid section of this type should look like — grid shape, required item semantics, where it applies.

3 Sources & status

Which borrowed entity types and status sets the type depends on. Borrowed, never duplicated.

4 Not for

Nearby patterns that should not be collapsed into this type. The guardrail against type drift.

Each convergence type should be treated as a reasoning contract. Pick the wrong one and the doc lies to you.

Rebuilt from the registry and the local catalog.

Run from the repo root to rebuild the overview HTML from the current type registry and living-doc catalog.

Command Default

$node scripts/render-registry-overview.mjs

Optional — point to a different catalog file for this run:

$LIVING_DOC_CATALOG_PATH=/path/to/living-docs.json \ node scripts/render-registry-overview.mjs

What must be in place

  1. scripts/living-doc-registry.json contains the current convergence types.
  2. The living-doc catalog at /Users/rene/.gtd/living-docs.json contains the docs you want sampled.
  3. Each catalog entry points to a rendered .html living doc with an embedded doc-meta block.
  4. For JSON source links, the cataloged HTML has a sibling .json file with the same basename.
