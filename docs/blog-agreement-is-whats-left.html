Agreement is what's left: living docs in a world of capable agents.

As AI capability expands, the bottleneck on complex work stops being capacity to execute. It becomes capacity to agree on what to execute — and that is a social problem, not a technical one.

RL
Rene Luijk Triadflow
PublishedApr 19, 2026 Read10 min StatusLive Filedocs/blog-agreement-is-whats-left.html

The easy wrong answer is that as models get smarter, the structured artifacts around them dissolve. Why maintain a living doc if a future agent can re-derive every projection on demand from Figma, the repo, Slack, and the ticket tracker? Why pin convergence types in a registry if a smart enough system can infer the right shape for each doc from first principles? The reasoning feels clean: more intelligence, less scaffolding.

This reasoning is wrong, but in a revealing way. It confuses capacity to synthesize with capacity to agree. The first scales with intelligence. The second doesn't. And the second is what complex work actually waits on.

What the system is actually betting on

Living-doc-compositor is often read as a projection tool — a view that converges Figma, code, tickets, and decisions into one structured page. That reading is accurate but shallow. The projection is the visible surface; the bet lives underneath.

The real bet is that a typed, named, portable ontology for converging work becomes the coordination substrate between humans and agents as both proliferate. Entity. Edge. Scope. Convergence type. Invariant. Objective facet. That isn't UI vocabulary. It's the grammar in which strategy and state get pinned down across time, across sessions, across team members, and — increasingly — across agents.

Which means the right test for this kind of system is not "does it help a human see the work" but "does it give humans and agents something to agree on." The first test is satisfied by any dashboard. The second is harder, and the second is what matters.

A concrete case: the archaeology tax

An example makes the bet less abstract. Consider what happens when someone engages seriously with an open-source issue — say, Textual #5380, a real mouse-input bug that's been sitting open with the maintainer saying "this is VS Code's bug, raise it there," a reporter pushing back with narrowed-down evidence, a workaround already shipped in a completely different repo (inspect_ai), four related issues forming a problem family around it, and a specific pair of functions in linux_driver.py that everyone ends up pointing at.

Before you can write code, you read. The issue body, six comments, the reporter's counter-arguments, the commit in the other repo, the four related issues, the 12-line function in linux_driver.py, the four ANSI escape codes being written there, and which of them xterm.js actually supports. That reading is an archaeology tax. Step away for a week, pay it again. Hand the issue to a new contributor, they pay it from scratch.

When you hand the same issue to Claude Code or Codex, the agent performs the same archaeology. It reads the thread, greps the driver file, follows PR references, maybe misses the inspect_ai workaround if you didn't paste the URL. The session ends, the context closes, and the next session starts cold. Every time. In tokens instead of minutes, but the work is the same. Smarter agents do this reading faster. They don't stop doing it.

A living doc for #5380 is where that tax gets paid once and deposits into a structured artifact: three symptoms with repro steps, four code anchors pinned to a commit SHA, three attempts including the shipped workaround, four orbit issues with typed relationships, four maintainer stances, one finding. Every piece already on GitHub or in the code — what's new is that it's one page, typed, pinned, and loadable. A fresh session opens the doc and starts in "where does the fix go?" mode, not "what is this issue?" mode.

Every abstract claim below has a concrete referent here. When the essay says shared compression of the search space, this is what compression looks like. When it says history of choices that cannot be re-derived from sources, the maintainer's stance and the reporter's pushback are that history. When it says commitment, not comprehension, the load-bearing fact that the ANSI codes are idempotent on xterm.js — the claim that makes the workaround safe to ship — is the commitment.

Four things that get more valuable, not less, as AI matures

1. Shared compression of the search space

A smarter agent with a bigger context window can, in principle, read anything. In practice, unbounded reading is expensive, non-reproducible, and produces subtly different conclusions on every pass. A living doc is a named, bounded subgraph. It says: this is what this piece of work is, these are its sources, this is its convergence shape.

The value of that compression grows with capability, not shrinks. A more capable agent executes deeper per unit of context. Handing it a projection that already knows the scope means the depth goes into reasoning about the work, not rediscovering what the work is.

2. Type contracts as agent guardrails

When an agent suggests, the cost of a bad output is a human saying no. When an agent executes, the cost is the actual mutation landing in the world. As execution becomes more reliable and more autonomous, the failure mode shifts from "wrong suggestion" to "unauthorized state change."

The useful lever here is not giving the agent more freedom. It is giving it clearer contracts. The convergence type system says: this section's cards have these fields, these status values, this structural contract. The validator rejects patches that violate the contract. The patch schema constrains the vocabulary of mutations to a fixed set. None of that is anti-intelligence. It's the shape of high-trust delegation.

Put differently: as capability scales, the boundary of what an agent can touch becomes the high-leverage design decision. Type contracts are where that boundary is drawn.

3. Portable artifacts outlive the tools that made them

A living doc is a single self-contained HTML file. No backend, no account, no vendor, no specific agent, no specific IDE. As AI proliferates across providers, stacks, and plug-ins, the artifact that stays readable anywhere is the coordination unit that survives platform churn. You can email it. Drop it in Slack. Open it offline. Load it into whatever agent you currently trust.

This matters more, not less, as the tooling layer fragments. A projection locked into a specific vendor is hostage to that vendor's roadmap. A projection that travels with its own UI and its own ontology is durable in a way a SaaS dashboard never is.

4. Governance becomes the high-leverage surface

When execution is cheap, deciding what to execute becomes the scarce resource. Deciding means: what's the objective, what invariants hold, what counts as done, what's the shape of the work. That deciding already happens — silently, in prose, in meetings, in implicit conventions that accrete over time. The living doc system is betting that making it explicit, typed, and checkable is a superpower.

The convergence type registry is where strategy is expressed. "This is a proof ladder" compresses more than ten pages of prose about what the work requires. "This is a capability surface" tells an agent the difference between updating a status and rewriting the scope. The ontology isn't decoration around the real work. As AI matures, the ontology becomes the real work that humans do.

"I can re-derive a view" is an AI capability. "There is a shared view we operate within" is a social artifact.

— The load-bearing claim

The load-bearing claim

The first scales with intelligence. The second does not.

Coordination is not primarily a synthesis problem. It's a commitment problem. A team of ten humans plus five agents, each individually brilliant, does not automatically converge on what matters just because each member is capable. They need a shared frame: what's the objective, what have we decided, what are we actually tracking. Without that frame, more capability produces more divergent outputs faster — agent stampedes over the same ambiguity, mutations that step on each other, decisions re-opened on every pass because nobody remembers they were decided.

The living doc is where those commitments live. Not because the doc itself is intelligent, but because it is where agreement gets pinned down and preserved. The registry is where the vocabulary of agreement is stored. The fingerprint is where the structural commitment is hashed. The invariants are where the rules of the game are named.

As agents proliferate, the arbitration surface becomes load-bearing. Without it, the system gets faster and less coherent. With it, the system gets faster and more coherent. That is the entire bet.

Where human leverage concentrates

Today, the human role in a living doc is roughly fifty-fifty. Humans both author sections (fill cards, write notes, update status) and govern the shape (define convergence types, set invariants, name objective facets). The AI-pass palette is a prosthesis for the first half — a way to lean on models for the filling work while humans retain judgment on the shape.

As agents mature, that balance inverts. The filling work becomes a standing process — the doc stays synced to its sources continuously, drift becomes a notification rather than a manual check, status transitions happen as soon as the underlying evidence lands. Humans stop filling. They govern.

SurfaceHuman role todayHuman role as agents mature
Sections and cards Authoring, updating, noting Reviewing, occasionally correcting
Convergence type registry Added rarely, by power users Primary surface of human work
Invariants and objective facets Optional, often skipped Central — the contracts agents operate within
AI-pass palette (Cmd+K) Human-initiated touches per card Transitional form — gives way to standing processes

This is the compression of human labor that the system is betting on. Humans do the rare, high-judgment ontology work. Agents do the continuous, high-volume maintenance work. The registry is the constitution both sides operate under.

The honest existential risk

What could make this bet wrong?

The strongest version of the "scaffolding" critique goes like this: if agents become fluent enough to re-derive any projection on demand, the durable artifact feels like tech debt. Every living doc is a crystallization of reasoning that a sufficiently capable agent can produce fresh every time, in the moment, tailored to the question being asked.

This critique has a real point. Many projections today are scaffolding — they exist because the models of 2024 couldn't hold the whole picture in context. For those cases, yes: more capability dissolves the need.

But the critique misses what durable docs actually preserve. A living doc is not primarily a view. It is a record of choices made. Attempts tried. Paths ruled out. Invariants held. Decisions ratified. Facets named. Coverage declared. That state is not derivable from sources because it is not a projection of the present — it is the history of how the present came to be. The sources tell you what the code does today; they don't tell you which of the eight plausible approaches were tried and rejected, or which invariant was added after a specific incident, or why the convergence type chosen for this section is right for this doc and not for the one next to it.

That history compounds. An agent in 2028 reading a living doc that has accumulated three years of choices gets handed something it cannot synthesize from sources. And as the pace of work accelerates, the gap between what's true now and what's recoverable from history widens. Living docs bet that the history matters more as intelligence expands, not less, because the history is where commitment lives, and commitment is not a synthesis problem.

The Textual case makes the counter concrete. A 2028 agent reading the live thread will still read the same comments, the same sibling issues, the same linux_driver.py. It can synthesize a current view. What it cannot synthesize is the record that the maintainer originally said "not ours," the reporter pushed back, a workaround shipped in inspect_ai, the idempotency claim was verified against xterm.js — because those are choices, not facts. They're only recoverable by re-doing the archaeology, one agent at a time, every time. Doing it once and preserving the result isn't scaffolding. It's the difference between a project accreting knowledge over years and resetting it on every session.

What this implies for how we build

If the bet is right, a few things follow.

The registry is the product. Convergence types, invariants, status sets, entity types — these are the durable artifact. The compositor UI, the AI-pass palette, the renderer, the mobile app: these are all replaceable surfaces over an ontology that outlives them. Investment in the registry compounds. Investment in UI decays.

Engine parity becomes load-bearing. Different agents will own different actions at different times. The shape of what an action does — what it reads, what it mutates, what warnings it emits — must be registry-declared, not locked into one engine's prompt. This is why the registry-driven action contract is worth building even though today the prompt-per-engine approach works.

The Cmd+K palette is a transitional form. It's the right interaction surface for today's reliability level — human-in-the-loop, explicit invocation, reviewed diff. As the contracts tighten, the palette gives way to standing processes: actions that fire when their preconditions land, propose patches that queue for review or auto-apply within a guardrail, and surface only when something needs human judgment.

Starter templates are inheritance. "Ship a feature," "prove a claim," "run support ops," "write a book" — each template is a compressed recipe for how to structure a class of work. That recipe transfers across teams, across decades, across agent generations. The templates are the compounding artifact of collective experience. Every time someone improves a template, every future user of that template inherits the improvement.

The deeper shift

As AI matures, the value doesn't move from substrate to AI. It moves from filling the doc to governing what the doc means. The registry becomes the high-leverage surface. Humans do the ontology work; agents do the projection work. The doc itself is the durable record of choices that survives every session, every agent, every model generation.

Cmd+K is a transitional form. The substrate is the point.

The question that stays open

None of this is a prediction. It's a position. The honest uncertainty is about timing and about edge cases: how long before standing processes replace palette interactions, which classes of work genuinely benefit from durable projections and which don't, where the governance layer becomes heavy enough to discourage contribution.

What we're fairly sure of is the direction. Coordination doesn't get easier as agents get smarter — if anything, it gets more important, because the cost of uncoordinated execution scales with execution speed. A typed, named, portable substrate where humans and agents pin down what matters is the thing that stops a world of capable agents from becoming a world of capable stampedes.

That's what living docs are for. Not to be clever. Not to look good. To be the place where agreement holds, across time and across minds, while everything else underneath it accelerates.
