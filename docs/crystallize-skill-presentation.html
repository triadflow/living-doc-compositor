Skill

/crystallize

Second-pass governance for living docs. Move silent reasoning into structured data inside the document.
Derives facets · coverage · invariants · rationale Stamps a metaFingerprint Not an authoring gate
What it does

Every LLM pass re-derives the same reasoning silently. Crystallize captures it.

A living doc's objective, invariants, and coverage exist implicitly — rebuilt by every reader, every agent, every review. Crystallize moves that reasoning out of the reader's head and into the document's structure.

Run it after a doc's shape has stabilized, not while sections are still being discovered. If there is no objective, the doc is too early — crystallize will stop and say so.
Derivations

Five fields written to the doc root.

Each one traceable to content that already lives in the doc. Meta is derived, never invented.

01

objectiveFacets

Decompose the free-form objective into typed, named facets (~5–10).

02

coverage

Edges from each facet to the sections and cards that carry it.

03

invariants

Observed regularities the doc already follows — not new contracts.

04

section.rationale

One sentence per section: why this convergenceType fits this content.

05

metaFingerprint

SHA pin over the current sections. Written last, after governance fields.

{ "objectiveFacets": [{ "id": "ui-fidelity", "name": "UI fidelity", "description": "..." }], "coverage": [{ "facetId": "ui-fidelity", "sectionId": "mobile-capabilities", "cardId": "auth-sign-in" }], "invariants": [{ "id": "capability-needs-code-path", "appliesTo": ["mobile-capabilities"] }], "metaFingerprint": "sha256:7c4a…" }
Usage

Four invocations. One idempotent mode. One safe-to-try mode.

/crystallize
Interactive. Picks the most relevant doc.
/crystallize <path/to/doc.json>
Crystallize a specific doc.
/crystallize <path> --refresh
Idempotent re-run. Preserves author-edited meta.
/crystallize <path> --dry-run
Print proposals without writing to the file.
Execution · 1–4

Read the doc. Propose the meta.

1

ReadLoad JSON, inspect every section and card

Parse objective, successCondition, and each section's id, title, convergenceType, cards. No objective → stop.

2

ProposeobjectiveFacets

Decompose the objective sentence into typed facets. Enumerated nouns and section titles are strong candidates. Target ~5–10. Fewer than 3 or more than 15 signals under- or over-specification.

3

Proposecoverage edges

For each facet, emit {facetId, sectionId, cardId} only where card content clearly relates. A facet can be carried by many sections. Missing carrier? Flag as orphan, don't fabricate.

4

Detectinvariants

Observe patterns the doc already follows — every capability card has codePaths, every verification card has a ticket. Zero invariants is a valid outcome.

Execution · 5–8

Write rationale. Confirm. Stamp. Re-render.

5

Proposeper-section rationale

One sentence per section explaining why this data wanted this convergenceType. Not the registry description — the contextual fit.

6

Writeshow summary, ask to accept or edit

Print facets, coverage edges, invariants, rationale counts, orphans, drift. On accept, write root fields in order: objective → objectiveFacets → coverage → invariants → metaFingerprint → sections.

7

StampmetaFingerprint

Compute via computeSectionFingerprint(doc.sections) from scripts/meta-fingerprint.mjs. Write last so it pins the exact sections inspected.

8

Re-rendernode scripts/render-living-doc.mjs <path>

The coherence-map section picks up derived items automatically. The freshness banner disappears.

Modes

Three ways to run. Each preserves something different.

default

Interactive propose → accept

Shows a full summary: facets, coverage, invariants, rationale counts, orphans, drift. User accepts, rejects, or edits before write.

--refresh

Idempotent re-run

  • Uses existing facets, invariants, rationale as priors
  • Recomputes coverage from current sections
  • Preserves ids unless explicitly removed
  • No-ops if nothing changed: "meta is current."
--dry-run

Inspect without writing

Prints all proposals. Reports what would change. No file modification. Safe to try on any doc to see what crystallization would propose.

Key principles

Five rules that shape every crystallization.

1

Meta is derived, not invented

Every facet, edge, invariant, and rationale must trace to something already in the doc.

2

Preserve author intent on refresh

Explicit edits to rationale, invariants, or facet descriptions survive re-crystallization unless structure forces a change.

3

Surface orphans, don't paper over them

An orphan facet is a signal — either the objective has a gap or the sections do. Either way, show it.

4

Stamp the fingerprint last

It pins the meta to the exact sections that were inspected. Write it after everything else is in place.

5

Crystallize when the shape is stable

If the user is still shaping the doc, say so and wait. Premature crystallization bakes in the wrong structure.
