Berhenti membayar biaya tersembunyi dari pekerjaan yang terpecah.

Produk Anda bukan masalahnya. Konteksnya yang jadi masalah.

Desain ada di Figma. Code di repo. Keputusan di Slack. Setiap tugas dimulai dengan cara yang sama: merekonstruksi apa yang sedang terjadi.

Manusia melakukannya. Agent melakukannya. Anda membayarnya dengan waktu, kesalahan, dan token.

Living doc mengubah realita yang tersebar itu menjadi satu tampilan terstruktur. Lengkap, terkini, langsung siap pakai. Untuk manusia dan AI.

Biaya

Anda sudah membayar untuk fragmentasi ini. Anda hanya tidak melihat tagihannya.

Setiap fitur tersebar di berbagai sistem. Setiap kali seseorang menyentuhnya, mereka membangun ulang gambarannya. Biaya itu terus bertumpuk.

T

Untuk tim Anda

Rekonstruksi, bukan eksekusi
  • Pekerjaan dimulai dengan rekonstruksi, bukan eksekusi
  • Context switching memperlambat setiap tugas, bukan hanya yang kompleks
  • Onboarding bergantung pada pengetahuan yang tak tertulis
  • Review kehilangan kedalaman karena gambaran utuhnya tidak terlihat
AI

Untuk AI stack Anda

Penemuan ulang, bukan penalaran
  • Token habis untuk menemukan ulang, bukan untuk bernalar
  • Tidak ada kontinuitas. Setiap sesi dimulai dari nol
  • Output terbatas pada sebagian kecil context window
  • Jawaban generik ketika struktur domain tidak ada
Hasilnya: Anda sudah membangun sesuatu yang bernilai. Tapi perilakunya masih seperti prototipe.
Pergeseran

Berhenti menulis dokumen. Mulai memproyeksikan realita.

Dokumen tradisional gagal karena satu alasan: mereka butuh pemeliharaan. Dan pemeliharaan selalu kalah.

Living doc membalikkan modelnya. Anda tidak menulis ringkasan. Anda mendefinisikan tampilan terstruktur di atas apa yang sudah ada.

Code tetap di code
Desain tetap di Figma
Tiket tetap di tracker Anda

Dokumennya menjadi lapisan semantik yang tipis:

Selalu selaras dengan realita
Selalu terstruktur
Selalu terbaca oleh manusia dan agent
Dokumen ini bukan deskripsi. Ini adalah antarmuka ke pekerjaan.

Kepercayaan

Kalau dokumennya tidak bisa dipercaya, percuma.

Living doc tidak berpura-pura menjadi kebenaran. Mereka mengungkapkannya.

✓

Cakupan yang eksplisit

Apa yang tercakup, dan apa yang tidak, terlihat di halamannya sendiri.

✓

Asal yang kanonis

Setiap elemen terhubung kembali ke sumbernya. Tidak ada fakta yang terputus.

✓

Timestamp yang presisi

Presisi penuh, bukan perkiraan. Anda tahu persis kapan snapshot diambil.

✓

State yang jujur

Dokumennya merefleksikan satu momen dalam waktu, bukan state "live" yang palsu.

Untuk agent

Akhirnya: sesuatu yang benar-benar bisa dipakai agent.

Agent gagal bukan karena bodoh. Mereka gagal karena tidak ada struktur.

Living doc memberi mereka yang dibutuhkan:

ScopeApa yang ada dalam bingkai
HubunganBagaimana semuanya terhubung
State saat iniApa yang benar sekarang

Jadi mereka bisa bernalar alih-alih menemukan ulang.

Dilengkapi paket skill untuk Claude Code. Setiap skill adalah prompt singkat yang mengajari agent cara membaca, memperbarui, atau memperluas living doc.

Mengapa

Saya tidak membangun ini untuk mendokumentasikan pekerjaan.

Saya membangunnya karena bekerja dengan Claude Code, OpenAI Codex, dan sebuah tim mengekspos pajak yang terus-menerus: setiap sesi dimulai dengan membangun ulang konteks.

Bukan menyelesaikan masalahnya. Merekonstruksinya.

AI tidak menghilangkan kompleksitas. Ia memperkuatnya. Lebih banyak pekerjaan paralel, lebih banyak keputusan implisit, lebih banyak state yang tersebar.

Jadi bottleneck-nya bergeser: dari eksekusi → ke pemahaman.

Ini ada untuk melindungi lapisan itu, supaya manusia dan agent bisa fokus pada masalahnya sendiri, bukan pada menemukannya ulang.
