Workflow

What the operator flow looks like now

The publishing track is now a sequence of checked operations instead of a memory exercise spread across conventions.

Open dossier index
1 / substrate

Research living doc exists

The living doc JSON/HTML is the structured substrate: docId, periods, sections, public URL, and current state.

2 / freeze

Dossier piece is written

The dossier HTML owns publication facts: title, period, frozen source commit, published date, summary, and paired living-doc reference.

3 / preflight

Pair checks and fact-check prep

Pair health validates the dossier contract. Fact-check prep surfaces blocking problems, warnings, and drift after the frozen commit.

4 / publish

One command refreshes the app surfaces

Publishing updates dossier meta, refreshes or skips the living-doc change-log according to the canonical-piece policy, refreshes the strip and index, and regenerates the registry.

5 / registry

The dossier app sees one clean state

living-doc-dossiers/v3 becomes the machine-readable surface for paired docs, pieces, cadence, source classes, and sync ownership.

Guardrails

The rules the system now enforces

The point of the new registry and workflow manifest is not more bureaucracy. It is to turn quiet drift into visible failures before publication.

Hard errors
  • Every dossier piece must resolve to exactly one living doc.
  • Dossier-backed docs must not overlap with the general registry.
  • Required dossier meta fields must all be present.
  • (livingDocId, period) must stay unique.
  • dossier-source-commit must resolve in git.
  • The living doc must declare the dossier period in periods[].
  • The public living-doc URL on the dossier must match the paired doc.
  • Every paired doc must have workflow metadata with cadence, source classes, and a valid sync mode + skill.
Warnings Policy
  • Missing living-doc change-log only matters for the latest published dossier piece.
  • Older dossier pieces do not get to overwrite the canonical living-doc change-log.
  • Older pieces rely on their own inline drift aside as the authoritative drift surface for that piece.
  • Missing fact-check pass is still treated as an operator warning surface.
