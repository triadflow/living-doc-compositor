Skills as Deep Repair for Objective Drift

The living doc already has enough governance structure to hold objective pressure. The missing move is inference-time repair: skills that treat invariants like conservation laws and force every completion, deferral, pivot, and proof claim to preserve the original objective.

Core shift

  • Do not add a heavier governance schema first.
  • Use chemistry-like laws to define what must be conserved.
  • Make skills responsible for detecting unstable document states.
  • Let repair skills update doc, tickets, and work state together.

The Stance

The governance layer is not weak because it lacks fields. It becomes weak when the inference pass treats its artifacts as permission to close.

A living doc skill should not ask "does the document contain completion evidence?" first. It should ask "what must be conserved across this transformation, and did the current proof conserve it?"

Where The Invariant Holds

The invariant sits above sections and proof cards. It governs transformations between states: seeding, composing, proving, deferring, pivoting, repairing, and closing.

Conserved quantity

The original objective, success condition, and non-negotiable constraints must remain traceable after every operation. If they change, that is a pivot reaction, not completion.

Violation

A proof card, closed issue, rendered artifact, or passing test substitutes for the objective and silently narrows what "done" means.

Governance Laws For Skills

These are not new document fields. They are operating laws a skill applies when reading the existing governance surface.

L1

Objective Conservation

Law
The original objective cannot disappear; unresolved mass must reappear as complete, blocked, deferred, pivoted, or follow-up.
Skill test
Compare objective terms to current proof, commits, tickets, generated artifacts, and final claims.
Repair
Rewrite the state honestly and reopen/create issues where objective mass was hidden.
L2

Activation Energy

Law
Closure requires a threshold event, not accumulated administrative heat.
Skill test
Check whether proof tried to fail against the objective, not only whether tests passed.
Repair
Downgrade closure to partial when the proof did not cross the objective threshold.
L3

Equilibrium Shift

Law
New evidence changes pressure. A previously stable state can become unstable.
Skill test
Read user rejections, new tickets, fresh commits, stale fingerprints, and changed source surfaces.
Repair
Rebalance the living doc: update stage, proof strength, decisions, blockers, and issue state.
L4

Catalysis

Law
A skill lowers the cost of the right transformation; it does not replace the objective.
Skill test
Check whether the skill is accelerating repair or merely making closure easier to justify.
Repair
Emit concrete doc patches, issue changes, and next work items instead of conceptual summaries.
L5

Reaction Pathway

Law
State transitions have allowed paths. Partial proof cannot jump directly to completion.
Skill test
Inspect whether stage, proof, issue state, and success condition all support the transition.
Repair
Move the doc to the nearest honest stage and name the missing transition step.
L6

Rate-Limiting Step

Law
The slowest unresolved objective constraint controls completion, not the largest pile of done work.
Skill test
Find the one remaining condition that would make a user rejection still valid.
Repair
Promote that condition to blocker or next work, even if many other cards are ready.

Skills That Move In This Direction

Each skill uses the existing living doc governance and source systems. The output is not advice. The output is a repaired operating state.

objective-conservation-audit

Find where the objective mass went.

Runs when completion is claimed or challenged. It maps every phrase of the objective and success condition to completed proof, blocker, deferral, pivot, or follow-up.

  • Fails closure when any objective phrase has no accounted state.
  • Writes missing mass back into the living doc and issue tracker.
activation-energy-review

Test whether closure crossed the real threshold.

Inspects whether final proof survived contradiction: source state, tests, rendered docs, issue status, and likely user rejection.

  • Distinguishes evidence accumulation from threshold crossing.
  • Can downgrade "complete" to "partial" without changing the objective.
equilibrium-rebalance

Repair the doc when new evidence shifts pressure.

Runs after a user challenge, new commit, reopened issue, stale fingerprint, or contradiction between source artifacts and proof cards.

  • Updates stage, status snapshot, proof strength, decisions, blockers, and rendered HTML.
  • Keeps the old state visible as history instead of pretending it never happened.
reaction-path-validator

Prevent illegal stage jumps.

Checks whether the proposed transition is allowed by the living doc's convergence types, relationships, invariants, proof state, and issue state.

  • Blocks direct jumps from useful slice to final closure.
  • Names the nearest honest next stage and its required operation.
catalytic-repair-run

Do the smallest real repair that restores objective fidelity.

Combines governance reading, source inspection, doc patching, rendering, issue updates, and verification into one repair pass.

  • Useful when an agent has started defending a bad closure claim.
  • Ends with an actual source-system state, not a conceptual explanation.

Deep Repair Run Shape

The chemistry metaphor becomes useful only when it changes the run protocol.

Input

Unstable claim

A completion, deferral, pivot, or proof claim feels plausible but may have lost the objective.

->
Skill reaction

Apply laws

Conservation, activation threshold, equilibrium pressure, pathway validity, and rate-limiting blockers are checked.

->
Output

Repaired state

The living doc, rendered HTML, issues, and next work all say the same honest thing.

Invocation Signals

These are the moments where normal implementation should pause and a deep repair skill should run.

User rejection

The user says the objective is not reached, the answer is slippery, or the agent is narrowing the objective after the fact.

Closure heat

Many closure artifacts appear at once: commits, rendered pages, green tests, closed tickets, final proof language.

Governance mismatch

Invariants, objective facets, coverage, rationale, source evidence, or issue state point in different directions.

Decision Output

A repair skill should end in one of five states. Anything softer invites the same drift back in.

Continue

The objective is still open and the next operation is known.

Repair

The doc or issue state was dishonest relative to source reality and has been corrected.

Block

A rate-limiting condition prevents closure or progress until resolved.

Defer

A remaining part is outside the current objective, with a trigger for reopening.

Pivot

The objective changed and the old objective is not claimed as complete.

Close

The conservation audit and activation review both fail to break the completion claim.

The Practical Governance Rule

This is the small rule that could be carried into skill instructions.

Before a skill may close, defer, or pivot a living doc objective, it must account for every conserved objective term as completed, blocked, deferred with trigger, pivoted with rationale, or explicitly outside the objective.
