Working Formulation

Living docs as shared, access-tolerant reasoning spaces

This page captures a central formulation for the living-doc system: it is not only for sharing status, and not only for generating typed documents. It is for creating a common reasoning surface around organizational objectives, even when different people and local AIs do not share the same source access.

Shared A common surface for management, implementers, and collaborators
Access-tolerant Useful even when source visibility differs across actors
Reasoning space Supports joint interpretation, planning, and coordination
Objective-linked Anchored in what the organization is trying to achieve

The formulation

It is for creating shared, access-tolerant reasoning spaces around organizational objectives

This is stronger than “dashboard”, “report”, or “documentation layer”. It describes the living doc as a place where differently situated actors can think together about the same work.

What happens in practice today

A living doc is shared with team members and management as an overview of the current state of affairs. Some people have direct access to GitHub tickets and underlying systems. Others do not. Yet both can still reason productively on the same living doc, including with their own local Claude Code sessions.

same doc different source access local AI reasoning shared objective surface

Important implication

The living doc is already functioning as an organizational abstraction layer. It lets people reason together over a typed operational view without requiring identical permissions on the underlying systems.

What makes this different from ordinary reporting

  • The living doc is not just a passive status display.
  • It is a structured surface on which people and AI can interpret evidence together.
  • It allows reasoning to begin from the same typed view even when deeper source access differs.
  • It helps management, implementers, and partial-access collaborators stay aligned on the same objectives.

Why access-tolerant matters

In a real organization, people do not all have the same visibility. Some have ticket access, some only see summaries, some can inspect repos, some can inspect cloud systems, and some only see the resulting artifacts. If the living doc can still serve as a common reasoning surface across those asymmetries, it becomes strategically useful.

What the living doc provides to different actors

Not everyone needs the same access to reason from the same surface.

Actor May have Uses the living doc for
Management Partial source visibility Understand whether objectives are being reached and where coordination is needed
Implementers Deep repo and ticket access Connect operational detail to broader objective and shared direction
Collaborators Limited system access Reason over the shaped work surface without needing full raw source access
Local AI in a session Whatever the current runtime can reach Interpret, update, extend, and reason from the typed surface

What must be true for this to work well

  • Type semantics need to stay strong and explicit.
  • The living doc should show enough provenance that people know what they are looking at.
  • The document should make scope visible, so people know what it covers and what it does not.
  • Coverage limits should not be hidden when runtime access is partial.
  • The doc should remain useful as a reasoning surface even for actors who cannot drill into every source.

What should not happen

Wrong direction

The living-doc system should not turn into a second permission model or a hidden source proxy that pretends everyone sees the same truth. Its value comes from making joint reasoning possible despite asymmetry, not from erasing the asymmetry.

Why this matters for organizational objectives

Objectives are usually achieved through complex, distributed work. The people accountable for those objectives often do not have the same operational visibility as the people implementing the details. A living doc helps bridge that gap by creating a typed surface where objective-level reasoning and implementation-level evidence can meet.

Objective level Are we moving toward the intended organizational outcome
Operational level What is actually built, blocked, drifting, or unresolved
Reasoning level What should be interpreted, escalated, or done next

Stronger restatement

A living doc is not only a document generated from multiple sources. It is a deliberate operational surface that allows differently situated humans and local AIs to reason together about the same objective, even when their access to the underlying systems is unequal.
