Friction frontier

Invariants become gates.

The current frontier is not writing better principles. It is compiling living-doc invariants into executable gatekeepers for inference-time harnesses.

Invariant

A rule the proof claim must obey.

Gate

Code that refuses invalid progress.

Verdict

Typed result, not prose memory.

Blocker

Issue created when the gate fails.

Evidence

What the run is allowed to claim.

core claim

A governed inference-time harness proves not only the output, but that the declared system produced it through the declared route.

Why this is hard

The proof surface has more layers than the command line shows.

TrackAndBack exposes the real shape: an interactive session starts a standalone worker, which may start a nested headless inference run. Each layer can drift.

Drift modes

The dangerous failures are proof-boundary failures.

The system can become more correct while the proof claim becomes less honest. Gates exist to keep those two things separate.

Drift 01

Observer becomes actor

The operator starts by watching, then patches, reruns, or steers a substep inside the claimed run.

Drift 02

Wrapper masks inner failure

The top-level process continues, but the nested JSONL shows an early crash, no terminal result, or malformed proposal.

Drift 03

Invariant stays prose

The living doc says the rule, but no runner, log parser, dashboard, or evidence bundle enforces it.

Drift 04

Support evidence is overclaimed

A manual rescue proves the fix path exists. It does not prove the autonomous system completed the run.

Gate stack

The gates should match the proof boundary, not the codebase boundary.

Selection

Next eligible item

No custom source, claim, graph, authority, replay target, or parser API flags.

Authority

Governed access

Source fetch, AWS access, deploy, and replay resolve from governed configuration.

Inference

Terminal JSONL result

Nested headless logs must end cleanly or produce a typed blocker.

Autonomy

No interactive rescue

Manual patching or steering downgrades the run to support evidence.

Replay

Post-deploy proof

`contract_ok=true`, `parse_clean=true`, and zero residual errors/warnings for clean proof.

Evidence

Sanitized durable refs

No raw emails, secrets, local paths, or proof-local-only promotion identity.

Blocker

Stop on failed gate

Typed issue, owning layer, acceptance criteria, and no same-turn repair without approval.

Claim

Strongest allowed status

The proof claim is calculated from gate results, not inferred from intent.

Current loop

Right now the skill is doing work the runner should eventually internalize. That is acceptable as a frontier state, but it should be named honestly.

Evidence verdicts

The evidence bundle should say what the run is allowed to claim.

clean proof

Replay-proven clean

All gates passed. The run stayed standalone and produced durable sanitized evidence.

partial proof

Supportive but incomplete

Some layers passed, but another required gate is missing, partial, or not yet executable.

support evidence

Operator-assisted repair

The system improved, but the proof boundary was broken by interactive intervention.

blocked

Formal blocker

A gate failed. The blocker issue becomes the next work item.

unknown

Insufficient evidence

The wrapper returned something, but direct logs or required artifacts are missing.

Implementation roadmap

The next hardening move is to push Level 2 skill rules into Level 3 and Level 4 gates.

1 Log classifier

Build a parser for nested `codex exec` JSONL and runtime logs. It should emit terminal state, reason code, owning-layer hypothesis, proposal/applied-change state, and premature-error signals.

2 Runner gate

Make the standalone worker fail closed when the nested inference result is missing, malformed, blocked, or non-terminal. The runner should create or prepare the blocker payload.

3 Evidence contract

Write the strongest allowed claim into the evidence bundle: clean proof, partial proof, support evidence, blocked, or unknown. The living doc should consume that result.

invariant -> code gate -> typed verdict -> evidence bundle -> living-doc status

This is the practical definition of governed inference-time work.

Definition of done

The harness is mature when the proof boundary is self-reporting.

At that point the interactive session can still help, but it cannot accidentally launder itself into the proof.

The runner owns selection.The operator starts the process; the system chooses the item and governed authorities.
The log parser owns nested inference truth.The direct event stream decides whether headless inference completed, blocked, or failed early.
The blocker gate owns failure.Failed gates produce issue-ready blocker records and stop the run.
The evidence bundle owns the claim.Status is derived from gate results, not from the operator's confidence.
The living doc owns memory.It reflects the strongest allowed machine verdict and preserves the method lesson.
