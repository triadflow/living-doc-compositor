Skills that rebalance a living doc.

When a living doc drifts, the first move is not to add more schema. The stronger move is to invoke skills that read the existing governance layer, detect imbalance, and repair the document, tickets, and proof state together.

Rene Luijk Triadflow
DraftedMay 7, 2026 Read4 min StatusDraft

A living doc can have a strong governance layer and still drift during inference. The common failure is not missing structure. It is false closure: the agent sees green tests, rendered pages, closed tickets, or a plausible proof ladder and starts treating those artifacts as permission to narrow the objective.

That is why repair belongs in skills. A skill can enter the document at the moment the state is unstable. It can read the objective, invariants, convergence types, acceptance criteria, issues, commits, and proof claims as one system. Then it can decide whether the next move is continue, repair, block, defer, pivot, or close.

The convergence-type refactor is the clean example. Template-generation work produced real artifacts, but the accountable objective was broader: convergence types had to become the executable canonical source, registry JSON had to regenerate from them, MCP had to expose their contracts, and drift tests had to prove the surfaces stayed aligned. A skill that stops at "the generator works" has not conserved the objective.

The operating rule: before a skill may close, defer, or pivot a living-doc objective, it must account for every conserved objective term as completed, blocked, deferred with trigger, pivoted with rationale, or explicitly outside the objective.

Why skills, not more fields

The governance layer already carries the pressure: objective, rationale, invariants, acceptance criteria, proof, source references, and stage state. Adding fields can make that pressure more visible, but it does not make inference obey it.

The harder problem is transformation. A living doc is alive because it moves between states. A skill has to ask what is conserved across that movement. If the objective changes, that is a pivot. If proof narrows the objective, that is drift. If a ticket closes without satisfying the accountable term it represents, that is hidden mass.

The repair set

living-doc-balance-scan Classifies the imbalance first: objective drift, proof drift, governance contradiction, stale source state, or mixed-stage confusion.
objective-conservation-audit Maps every objective phrase to completed proof, blocker, deferral, pivot, follow-up, or unresolved gap.
activation-energy-review Checks whether closure crossed the real threshold, instead of only collecting closure-shaped artifacts.
equilibrium-rebalance Repairs the state when evidence, user rejection, issue state, or governance signals pull in different directions.
reaction-path-validator Blocks invalid stage jumps, especially from useful slice to final closure.
catalytic-repair-run Runs the repair as work, not advice: update the doc, render it, adjust issues, verify source state, and commit the change.

The chemistry lens

The chemistry language is useful because it keeps the skill focused on state change. Conservation says the objective cannot disappear. Activation energy says closure needs a real threshold. Equilibrium says the document, source system, proof, and issues must not contradict each other. A catalyst lowers the cost of the right transformation without changing what must be conserved. A reaction path says not every stage transition is valid.

Those are not decorative metaphors. They are checks a skill can perform against the current living doc. They make the repair operational.

What this does not solve

A repair skill cannot make weak source evidence strong. It cannot close an objective whose accountable terms are still unresolved. It cannot turn a user rejection into a success by adding more explanation. Its job is narrower: make the document honest enough that the next operation is valid.

What a repair should return

  • continueThe objective is still open and the next operation is known.
  • repair-firstThe document state is misleading and must be corrected before implementation continues.
  • blockedA rate-limiting condition prevents progress or closure until resolved.
  • deferred-with-triggerA remaining part is outside the current objective, with a clear reopening trigger.
  • pivotedThe objective changed, and the old objective is not claimed as complete.

A rebalance skill should not end in commentary. It should leave the living doc more honest than it found it: objective terms accounted for, invalid closure blocked, and the next operation visible.
