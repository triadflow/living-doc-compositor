Opening / lede

Start from the mismatch the refactor fixed

Current

The core shape is clear: convergence types are the ground-level semantic unit of a living doc. The living doc instance remains JSON because an instance is state. The registry JSON remains important because other surfaces consume it.

Proposed

The system had a quiet mismatch. Convergence types were already the way we named reusable document semantics, but the rest of the stack still had to treat much of that meaning as JSON vocabulary, rendered prose, or template-local convention.

That works until a skill needs to ask a sharper question: what fields does this section own, what status logic is valid, what proof does it expect, and what stage movement would be legal?

Argument
The proposed opening follows the project voice more closely: pressure first, then mechanism. It gives a reader a reason to care before introducing the source-layer distinction.
Risk
This is less compact than the current opening. It spends more words before the thesis, but those words make the architectural claim easier to enter.
Dek / index copy

Reduce surface-list density in the dek

Current

The reusable semantics of a living doc start at the convergence type. Code-defined convergence types make that existing ground level executable across the registry artifact, optional starter templates, MCP tools, rendered context, drift checks, and skill reasoning surfaces.

Proposed

A living doc stays JSON because it is state. A convergence type becomes code because it is a reusable semantic contract. Once that contract is executable, the registry, rendered docs, MCP tools, skills, and drift checks can all read from the same source.

Argument
The current dek is accurate but list-heavy. The proposed version uses the article's strongest closing sentence as the entry point and makes the distinction legible in two beats.

What this does not solve

Executable convergence-type contracts do not decide the objective. They do not prove that a living doc is current. They do not remove judgment from stage movement or repair. They make the rules inspectable enough that judgment has something stable to inspect.

Suggested revised article arc

Pressure: convergence types had meaning, but tools could not ask the type for its contract.

Concrete case: code definitions, generator check, registry JSON, rendered context, MCP contract access, and drift tests.

Distinction: living doc JSON is state; convergence type code is reusable semantics; registry JSON is generated artifact; starter templates are optional compositions.

Mechanism: skills and MCP tools retrieve contracts instead of reconstructing them from prose.

Boundary: executable contracts support judgment; they do not replace objective, proof, or stage judgment.

Rule: the architecture is done only when one semantic source produces the operational surfaces and tests prove they have not drifted.
